# Contributing

Titanbase is built in the open and contributions are welcome. This page covers setup and the contribution workflow.

## Prerequisites

- Node.js 20+
- pnpm 9+
- Git

## Setup

```bash
git clone https://github.com/titanbaserun/titanbase
cd titanbase
pnpm install
pnpm test       # run the test suite
pnpm dev        # start the editor in dev mode
```

## Ways to contribute

### Report a bug

Open an issue with a description, steps to reproduce, expected vs. actual behavior, and your `.titan.json` if relevant (you can anonymize table and column names).

### Suggest a feature

Open an issue tagged `feature-request`. Describe the use case first, then the proposed solution — we value understanding the *why* before the *how*.

### Write an exporter

Exporters are the easiest way to contribute. Each is a standalone package implementing the [Plugin API](/plugins). See `packages/plugins/postgres/` for a reference implementation.

## Submitting code

1. Fork the repository.
2. Create a branch from `main`: `git checkout -b feat/my-feature`.
3. Make your changes and add or update tests.
4. Run `pnpm test` and `pnpm lint`.
5. Open a pull request against `main`.

All PRs require review before merge, and history is kept clean with squash merges.

## Code style

- TypeScript strict mode
- No `any` — prefer `unknown` with type narrowing
- Functional style over classes
- Tests with Vitest; format with Prettier; lint with ESLint

## Commit messages

Follow conventional commits:

```
feat(core): add enum support to the IR type system
fix(pg): handle nullable UUID columns
docs: clarify many-to-many join tables
test(mysql): cover inline enum generation
```

# Titanbase Cloud

<Status value="Planned" />

Titanbase Cloud may become a future product layer for teams. **It is not required** to use the local editor, `.titan.json`, exporters, the planned desktop app, or the planned CLI.

<Callout type="note">
The local product is complete on its own. No account is required, nothing is uploaded, and the cloud is never a dependency of the local editor or the file format.
</Callout>

## What it is not

- It is **not** required to design schemas, validate, or export.
- It does **not** gate the local editor, `.titan.json`, or the exporters behind an account.
- There is no pricing, billing, or sign-up to document — those are out of scope for the open-source product.

## Possible future capabilities

If and when a cloud layer is built, these are the kinds of team features it might offer (all optional, none available today):

- Team workspaces.
- Realtime collaboration.
- Comments and review workflows.
- Version history.
- Live database import.
- Schema drift detection.
- Saved connections.
- Optional AI schema review.

These are exploratory ideas, not commitments or available features. The open-source, local-first product remains the focus, and the docs will not present any of the above as available until it ships.

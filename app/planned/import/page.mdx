# PostgreSQL SQL Import

<Status value="Planned" />

Importing an existing PostgreSQL `.sql` file into a `.titan.json` schema is on the roadmap. **It is not available yet.**

<Callout type="note">
Import is planned as <strong>local file import only</strong> — you point it at a `.sql` file. It will <strong>not</strong> connect to a live database, and it will not ask for credentials. Your schema stays local.
</Callout>

## What it will do

The planned importer will parse a PostgreSQL DDL file and produce a `.titan.json` you can open in the editor.

## Planned support

The initial target is a practical subset of PostgreSQL DDL:

- `CREATE TYPE ... AS ENUM` → enums
- `CREATE TABLE` → tables and columns
- Primary keys
- Foreign keys
- Unique constraints
- Indexes
- Comments → descriptions

Statements outside the supported subset will be reported as **import warnings** rather than failing the whole file.

## Not planned for import

- Live database connections.
- Reading credentials or connection strings.

Live database import is part of the much later, optional [Titanbase Cloud](/planned/cloud) thinking — not the local importer.

When import ships, this page will document the exact supported subset and switch to **Status: Available**.
